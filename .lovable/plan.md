

# Fix PWA Manifest for PWABuilder APK Packaging

## Root Cause

There are **two competing manifests** in the build:

1. `public/manifest.json` — a static file with relative URLs (`/player`, `/icon-512x512.png`)
2. VitePWA's auto-generated `manifest.webmanifest` — with absolute URLs

The `index.html` has a manual `<link rel="manifest" href="/manifest.json">` that points to the static file, so PWABuilder reads the static one and ignores VitePWA's generated manifest. This creates confusion and may cause PWABuilder's agent to fail when resolving relative URLs or detecting the service worker relationship.

## Plan

### 1. Delete `public/manifest.json`

Remove the static file entirely. VitePWA will generate the manifest automatically during build with all the correct metadata.

### 2. Remove manual manifest link from `index.html`

Remove `<link rel="manifest" href="/manifest.json" />` from the `<head>`. VitePWA automatically injects its own manifest link into the built HTML — the manual one is redundant and causes the conflict.

### 3. Fix icon `purpose` in `vite.config.ts`

Split the single `"any maskable"` icon into two separate entries (one `"any"`, one `"maskable"`). PWABuilder and Google Play require this separation for proper adaptive icon handling.

### 4. Update VitePWA manifest to use root-relative URLs

Change icon `src` and `start_url`/`scope` from absolute URLs (`https://glowhub-pixel.lovable.app/...`) to root-relative paths (`/player`, `/icon-512x512.png`). PWABuilder resolves these correctly against the manifest URL origin, and root-relative paths are more portable. Absolute URLs can cause issues if the domain changes or during PWABuilder's crawl.

### Files changed

- **Delete**: `public/manifest.json`
- **Edit**: `index.html` — remove the manifest link line
- **Edit**: `vite.config.ts` — fix icon entries and use root-relative paths

### Technical detail (vite.config.ts manifest section)

```typescript
manifest: {
  name: "GlowHub - Digital Signage",
  short_name: "Glow",
  description: "Upload, schedule, and push content to remote screens in real-time.",
  theme_color: "#00A3A3",
  background_color: "#00A3A3",
  display: "standalone",
  orientation: "landscape",
  start_url: "/player",
  scope: "/",
  icons: [
    {
      src: "/icon-192x192.png",
      sizes: "192x192",
      type: "image/png",
    },
    {
      src: "/icon-512x512.png",
      sizes: "512x512",
      type: "image/png",
      purpose: "any",
    },
    {
      src: "/icon-512x512.png",
      sizes: "512x512",
      type: "image/png",
      purpose: "maskable",
    },
  ],
}
```

After publishing, PWABuilder will see a single, clean manifest generated by VitePWA with proper icons and a working service worker — no more conflicts or stale static files.
